Заголовки

Поисковые роботы собирают из заголовков информацию для семантического анализа. Особое значение для ботов имеют заголовки уровней h1, h2, h3. Их непоследовательное или неправильное применение может негативно сказаться на ранжировании станицы.

Главным правилом в расстановке заголовков является их иерархия. Необязательно, чтобы теги шли друг за другом по размеру, но недопустимо в тексте использовать, например, h3 если у вас нигде не присутствует h2 или применять h6 без h5.

    Кратко:
  • На странице должен быть только один h1
  • h1 должен соответствовать содержанию страницы
  • Соблюдение иерархии (h1->h2->h3...)
  • h1 должен содержать соответствующие ключевые слова
  • текст в h1 не должен быть длиннее 120 символов
  • заголовки нельзя использовать в качестве ссылок
    Частые ошибки при составлении заголовков:
  • отсутствие h1
  • h1 состоящий только из одного слова
  • короткий h1
  • слишком длинный h1
  • h1 не соответствующий содержанию статьи

h1 и ключи

Помимо title страницы, основной заголовок (H1) является еще одним важным элементом оптимизации на странице. Как и в title, ключевые слова, используемые в этом заголовке, должны появляться и в текущем тексте страницы.

  • Если заголовок H1 содержит наиболее важные ключевые слова для страницы, текст страницы следует пересмотреть, чтобы ключевые слова действительно использовались на странице.
  • или если текст страницы был верно написан и оптимизирован, ключевые слова в заголовке H1 должны быть скорректированы таким образом, чтобы заголовок содержал ключевые слова, имеющие значение для страницы.

Важно: h1 и title не должны быть одинаковыми.

Description

Мета-тег description существует для описания страницы, которое помогает пользователю определиться, стоит ли переходить на страничку сайта или нет. Текст этого тега часто выводится поисковыми системами сразу под ссылкой на сайт. По данным экспертного анализа сайтов, в 60% случаев Google формирует сниппет из мета-тега description.

    При составлении description нужно учитывать:
  • description не должен быть длиннее 160 символов(с учетом пробелов)
  • description должен быть точным и понятным, побуждающим юзера к действию
  • description должен содержать ключевые слова
  • description должен описывать содержимое страницы
  • description должен быть уникальным и отличным от тегов h1 и title

Теги b & strong

Визуально теги b и strong выглядят одинаково - они делают текст жирным.

Но тогда как тег b лишь форматирует текст, strong несет в себе семантическую информацию, определяя "ВАЖНОСТЬ" текста.

    Используя эти теги стоит учитывать:
  • Текст в этих тегах не должен быть длиннее 70 символов
  • Количество этих тегов должно соответствовать длине статьи (если статья короткая кол-во тегов нужно сократить)

Title

Title веб-страницы является одним из основных элементов оптимизации поисковой системы. В поисковых системах, таких как Google, ключевые слова в Title страницы имеют жизненно важное значение для рейтинга страницы. Кроме того, Title отображается в качестве наиболее заметного элемента в результатах поиска, поэтому он должен побуждать пользователя перейти на страницу.

Поэтому правильный заголовок должен быть выбран разумно и в соответствии с содержанием страницы. В идеале заголовок отображает краткое и понятное резюме и содержит релевантные ключевые слова, по которым страница стремится достичь хорошего рейтинга в поисковых системах.

    Составляя Title нужно учитывать:
  • Длина title должна быть не более 55-65 символов
  • Важные ключевые слова должны быть в title
  • title должен содержать 2-3 ключевых слова
  • Самые важные ключевые слова должны появляться первыми в заголовке
  • Title не должен меняться слишком часто

Дублированный контент

Дублированный контент - это идентичный контент, который несколько раз встречается на разных страницах.

Страницы не обязательно должны быть идентичными на 100%, если заголовки страниц различаются, и обе страницы имеют одинаковое содержание или текст, тем не менее они будут отмечены как дубли. Дублированный контент является огромной проблемой для поисковых систем при индексации веб-контента, и поэтому его следует исключить, чтобы избежать негативного воздействия на сайт.

Размер статьи

Количество слов в статье должно быть не менее 500.

В статьях обязательно должно присутствовать хотя бы 2-3 параграфа (нельзя делать статьи состоящие только из списков).
